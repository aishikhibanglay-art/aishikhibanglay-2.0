PAYMENT FLOW:
POINT 1: SSLCommerz (Bangladesh):
Student clicks enroll → POST to /api/payment/sslcommerz/init → redirect to SSLCommerz payment page

Verify payment server-side → create orders row → create enrollments row → send receipt email via Resend

POINT 2: Stripe (Global):
Student clicks enroll → POST to /api/payment/stripe/checkout → Stripe Checkout Session
On success → Stripe webhook POSTs to /api/payment/stripe/webhook (Supabase Edge Function)
Verify webhook signature → create orders row → create enrollments row → send receipt email
